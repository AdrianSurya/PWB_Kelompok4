JavaScript Concepts

A comprehensive guide to JavaScript fundamentals

JS Errors

Errors in JavaScript include SyntaxError, ReferenceError, and TypeError, which occur due to syntax issues, undeclared variables, or wrong data types.

Example:

try {
  console.log(x); // x is not defined
} catch (err) {
  console.log(err.message);
}

JS Scope

Scope determines the accessibility of variables in JavaScript.

Example:

let x = 10; // Global
function test() {
  let y = 20; // Local
  console.log(x + y); // 30
}
test();

JS Hoisting

JavaScript hoists variable and function declarations to the top of their scope.

Example:

console.log(a); // Output: undefined
var a = 5;

JS Strict Mode

Strict mode enables stricter parsing and error handling in JavaScript.

Example:

"use strict";
x = 10; // Error: x is not defined

JS this Keyword

The this keyword refers to the current object in context.

Example:

const person = {
  name: "Alice",
  greet() {
    console.log(this.name);
  },
};
person.greet(); // Output: "Alice"

JS Arrow Function

Simplifies function expressions with concise syntax.

Example:

const add = (a, b) => a + b;
console.log(add(2, 3)); // Output: 5

JS Classes

Provides a syntax for creating objects and inheritance in JavaScript.

Example:

class Car {
  constructor(brand) {
    this.brand = brand;
  }
  display() {
    console.log(`Brand: ${this.brand}`);
  }
}
const myCar = new Car("Toyota");
myCar.display();

JS Modules

Helps to split code into reusable modules.

Example:

// file1.js
export const greet = () => console.log("Hello");

// file2.js
import { greet } from "./file1.js";
greet();

JS JSON

JSON is a lightweight data-interchange format.

Example:

const jsonData = '{"name": "John", "age": 30}';
const obj = JSON.parse(jsonData);
console.log(obj.name);

JS Debugging

Debugging involves identifying and fixing errors in code.

Example:

console.log("Debugging variable x:", x);

JS Style Guide

Follow conventions such as camelCase, proper indentation, and comments.

Example:

let firstName = "John";

JS Best Practices

Avoid global variables, use const or let, and prefer === over ==.

JS Mistakes

Common mistakes include undeclared variables and using == instead of ===.

JS Performance

Optimize by minimizing DOM manipulation and using efficient loops.

JS Reserved Words

Words like class, let, and return are reserved and cannot be used as identifiers.
